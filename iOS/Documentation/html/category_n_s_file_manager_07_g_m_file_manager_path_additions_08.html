Detailed Description

A few useful methods for dealing with paths.

Method Documentation

- (BOOL) gtm_createFullPathToDirectory: (NSString *)  path
attributes: (NSDictionary *)  attributes 

For the Unix-y at heart, this is "mkdir -p". It tries to create the directory specified by |path|, and any intervening directories that are needed. Each directory that is created is created with |attributes| (see other NSFileManager doco for the details on |attributes|).

If you are building for 10.5 or later, you should just use the new api: createDirectoryAtPath:withIntermediateDirectories:attributes:error:

Args: path - the path of the directory to create. attributes - these are defined in the "Constants" section of Apple's NSFileManager doco

Returns: YES if |path| exists or was able to be created successfully NO otherwise

- (NSArray *) gtm_filePathsWithExtension: (NSString *)  extension
inDirectory: (NSString *)  directoryPath 

Return an the paths for all resources in |directoryPath| that have the |extension| file extension.

Args: extension - the file extension (excluding the leading ".") to match. If nil, all files are matched. directoryPath - the directory to look in. NOTE: Subdirectories are NOT traversed.

Returns: An NSArray of absolute file paths that have |extension|. nil is returned if |directoryPath| doesn't exist or can't be opened, and returns an empty array if |directoryPath| is empty. ".", "..", and resource forks are never returned.

- (NSArray *) gtm_filePathsWithExtensions: (NSArray *)  extensions
inDirectory: (NSString *)  directoryPath 

Same as -filePathsWithExtension:inDirectory: except |extensions| is an NSArray of extensions to match.
