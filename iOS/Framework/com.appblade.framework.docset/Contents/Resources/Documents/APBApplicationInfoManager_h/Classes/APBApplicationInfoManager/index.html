hashExecutable

- (NSString*)hashExecutable;
Return Value

A MD5 hash file value of the executable

Discussion

we find the hash of the executable by first taking [NSBundle mainBundle] executablePath] and finding the hashFile value.

This should not be confused with the executableUUID call, which walks the load commands and does not return an MD5 hash

See Also

FileMD5HashCreateWithPath for how we find the MD5 hash of a file

hashFile:

- (NSString*)hashFile:(NSString *)filePath;
Parameters
filePath

the path to the file you'd like to know the hash of

Return Value

A MD5 hash file value of an arbitrary filepath, or the hash of an empty string if it does not exist

Discussion

The file we are finding the hash of should not be affected in any way by the Hashing function. If no file exists at the path, the hash of an empty string will be returned "d41d8cd98f00b204e9800998ecf8427e"

See Also

FileMD5HashCreateWithPath for how we find the MD5 hash of a file

hashInfoPlist

- (NSString*)hashInfoPlist;
Return Value

a uuid of the executable

Discussion

we find the uuid of the executable by using the symbol _mh_execute_header and walking the load commands of its program to determine the ending (or beginning) of any section or segment in the program.

See Also

FileMD5HashCreateWithPath for how we find the MD5 hash of a file

Properties

executableUUID

executableUUID

@property (nonatomic,
    strong) NSString *executableUUID;
Return Value

a uuid of the executable (00000-0000-0000-0000-00000000 if simulator)

Discussion

we find the uuid of the executable by using the symbol _mh_execute_header and walking the load commands of its program to determine the ending (or beginning) of any section or segment in the program.

The value of the link editor defined symbol _MH_EXECUTE_SYM is the address of the mach header

This should not be confused with the hashExecutable call, which returns an MD5 hash
